# Nexora - Política de Segurança

Obrigado por ajudar a manter o Nexora seguro.

A segurança dos nossos usuários é uma prioridade. Caso você descubra uma vulnerabilidade, pedimos que nos informe de forma responsável para que possamos corrigir o problema o mais rápido possível.

---

## Versões Suportadas

| Versão | Suportada |
| ------- | --------- |
| 1.0.x   | ✅ |
| Beta    | ✅ |
| Anteriores | ❌ |

---

## Como reportar uma vulnerabilidade

Caso encontre uma falha de segurança, envie um relatório contendo:

- Descrição da vulnerabilidade;
- Passos para reproduzir o problema;
- Evidências (prints, vídeos ou logs);
- Impacto causado;
- Sugestão de correção (opcional).

Enquanto não tivermos um e-mail oficial, utilize a aba **Issues** do GitHub para relatar problemas que não exponham dados sensíveis.

---

## Tempo de Resposta

Nossa meta é responder dentro dos seguintes prazos:

- Confirmação do recebimento: até 48 horas;
- Primeira análise: até 7 dias;
- Correção: conforme a gravidade da vulnerabilidade.

---

## Vulnerabilidades Cobertas

Aceitamos relatórios sobre:

- Falhas de autenticação;
- Vazamento de informações;
- SQL Injection;
- Cross-Site Scripting (XSS);
- Cross-Site Request Forgery (CSRF);
- Escalada de privilégios;
- Execução remota de código;
- Qualquer outra vulnerabilidade que comprometa a segurança do Nexora.

---

## Divulgação Responsável

Pedimos que nenhuma vulnerabilidade seja divulgada publicamente antes que uma correção seja disponibilizada.

Após a correção, os pesquisadores poderão ser reconhecidos pelos seus relatos, caso desejem.

---

## Agradecimentos

Agradecemos a todos que colaboram para tornar o Nexora uma plataforma cada vez mais segura.
